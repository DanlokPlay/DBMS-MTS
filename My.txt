2. Концептуальная модель базы данных
Таблицы и их атрибуты:

Клиент:

id_клиента (PK) — INT
имя — NVARCHAR(100)
тип_клиента — NVARCHAR(20) (Физическое или Юридическое лицо)
баланс — MONEY
номер_телефона — NVARCHAR(15) (UNIQUE)
id_тарифа (FK) — INT
Тариф:

id_тарифа (PK) — INT
название_тарифа — NVARCHAR(50)
тип_тарифа — NVARCHAR(20) (Корпоративный или Некорпоративный)
стоимость_перехода_на_тариф — MONEY
стоимость_минуты_по_городу — MONEY
стоимость_минуты_междугородний — MONEY
стоимость_минуты_международный — MONEY
Звонок:

id_звонка (PK) — INT
id_клиента (FK) — INT
номер_получателя — NVARCHAR(15)
время_звонка — DATETIME
длительность_разговора — INT
тип_соединения — NVARCHAR(20) (по городу, междугородний, международный)
стоимость_звонка — MONEY
Запрос_детализации_звонков:

id_запроса (PK) — INT
id_клиента (FK) — INT
дата_начала — DATE
дата_окончания — DATE
Связи между таблицами:

Клиент - Тариф (1-n): Один клиент может иметь один тариф, но один тариф может быть у нескольких клиентов.
Клиент - Звонок (1-n): Один клиент может сделать несколько звонков.
Клиент - Запрос детализации звонков (1-n): Один клиент может запросить несколько детализаций звонков.
3. Физическая модель базы данных
Таблицы и их атрибуты:

Клиент:

client_id: INT (PK)
name: NVARCHAR(100)
client_type: NVARCHAR(20) (Физическое или Юридическое лицо)
balance: MONEY
phone_number: NVARCHAR(15) (UNIQUE)
tariff_id (FK): INT
Тариф:

tariff_id: INT (PK)
tariff_name: NVARCHAR(50)
tariff_type: NVARCHAR(20) (Корпоративный или Некорпоративный)
switch_cost: MONEY
cost_local: MONEY
cost_long_distance: MONEY
cost_international: MONEY
Звонок:

call_id: INT (PK)
client_id (FK): INT
receiver_number: NVARCHAR(15)
call_time: DATETIME
call_duration: INT
call_type: NVARCHAR(20) (по городу, междугородний, международный)
call_cost: MONEY
Запрос детализации звонков:

request_id: INT (PK)
client_id (FK): INT
start_date: DATE
	: DATE
Связи:

Клиент - Тариф (1-n): Один клиент может иметь только один тариф, но один тариф может быть у нескольких клиентов.
Клиент - Звонок (1-n): Один клиент может сделать много звонков.
Клиент - Запрос детализации звонков (1-n): Один клиент может сделать несколько запросов на детализацию звонков.
